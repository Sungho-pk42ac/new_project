# 코드 리뷰 상세 체크리스트

리뷰 시 카테고리별로 참고하는 점검 항목 모음. 모든 항목을 기계적으로 적용하지 말고, 리뷰 대상 코드에 해당하는 것만 적용한다.

## 정확성 (Correctness)

- [ ] 경계 조건: 빈 배열/문자열, 0, 1개 원소, 최대값
- [ ] off-by-one: 반복문 인덱스, 슬라이스 범위
- [ ] null / undefined / NaN 처리
- [ ] 비동기: await 누락, race condition, 처리 안 된 Promise rejection
- [ ] 타입 강제 변환으로 인한 의도치 않은 동작 (`==` vs `===`)
- [ ] 조건문 분기 누락 (else, default)
- [ ] 반환값을 호출부에서 올바르게 사용하는지

## 보안 (Security)

- [ ] 사용자 입력 검증/이스케이프 (XSS, SQL/명령 인젝션)
- [ ] 하드코딩된 비밀정보(API 키, 비밀번호, 토큰) 노출
- [ ] 인증/인가 체크 누락
- [ ] 안전하지 않은 역직렬화, 경로 순회(path traversal)
- [ ] 민감 정보 로깅
- [ ] 의존성의 알려진 취약점

## 에러 처리 (Error Handling)

- [ ] try/catch 또는 에러 전파 누락
- [ ] 조용한 실패(에러를 삼키고 빈 catch)
- [ ] 사용자에게 의미 있는 에러 메시지 제공 여부
- [ ] 리소스 정리(파일 핸들, 연결, 타이머 clearInterval/clearTimeout)
- [ ] 실패 시 일관성 없는 상태로 남는지

## 품질 / 유지보수 (Quality)

- [ ] 중복 코드 — 재사용 가능한 함수로 추출 가능한지
- [ ] 함수가 너무 길거나 한 가지 일만 하지 않는지
- [ ] 과한 복잡도 / 불필요한 추상화 (YAGNI)
- [ ] 네이밍이 의도를 드러내는지
- [ ] 매직 넘버/문자열 → 상수화
- [ ] 죽은 코드, 미사용 변수/import
- [ ] 기존 유틸/패턴을 재사용하지 않고 새로 만든 부분

## 컨벤션 (Conventions)

- [ ] 프로젝트 CLAUDE.md 규칙 준수 (이 프로젝트: camelCase, 함수 JSDoc 한국어 주석, console 대신 로깅, 주석 한국어)
- [ ] 주변 코드의 포매팅/스타일과 일관성
- [ ] 주석이 "무엇"이 아닌 "왜"를 설명하는지
- [ ] 공개 API/함수에 문서 주석 존재 여부

## 프런트엔드 / HTML+JS 특화 (이 프로젝트 관련)

- [ ] DOM 조작 시 요소 존재 확인(querySelector null)
- [ ] 이벤트 리스너 중복 등록/누수
- [ ] localStorage 접근 시 try/catch (용량 초과, 비활성화 환경)
- [ ] innerHTML에 사용자 입력 직접 삽입(XSS) — textContent 사용 권장
- [ ] setInterval/setTimeout 정리
- [ ] 반응형/접근성(라벨, 대비, 키보드 조작)
